This repository contains documentation for the EDD Platform project. The docs are built with Mintlify and live under `docs/`.

## Preview the docs

Run Mintlify from the documentation root:

```bash
cd docs
npx mint dev
```

Mintlify usually serves the local preview at `http://localhost:3000`.

## Read the project in order

1. Start with [Eval-Driven Design](/concepts/eval-driven-design) to understand the product thesis.
2. Walk through the [Sentiment Observer demo](/guides/sentiment-observer-demo), the canonical proof-loop example.
3. Read the [Error Analysis Wizard guide](/guides/error-analysis-wizard) to understand how failure modes are built from live traces.
4. Read [Architecture overview](/architecture/overview) to see the workflow/control plane model and package boundaries.
5. Use the [Glossary](/reference/glossary) to keep terminology consistent.
6. Review [Local development](/guides/local-development) before running project commands.

## What not to assume

This docs site describes both current functionality and intended design. When a page describes planned behavior, it uses language such as "intended design" or "should." Do not treat those sections as proof that a production implementation exists.
