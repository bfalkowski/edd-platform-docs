---
title: "Scenarios and Eval Contracts"
description: "Define the cases agents must handle and the evidence required to judge them."
---

A `Scenario` describes the situation an agent must handle. It should be specific enough to reproduce the behavior being evaluated, but not so narrow that it only tests one memorized transcript.

An `EvalContract` defines how success is judged for a scenario or scenario set. It should describe expected behavior, constraints, scoring criteria, and required evidence.

## Scenario contents

A useful scenario often includes:

- User goal or task.
- Required context or fixtures.
- Relevant constraints.
- Expected high-level behavior.
- Known edge cases.
- Links to traces, datasets, or artifacts when available.

## Eval contract contents

A useful eval contract often includes:

- Success criteria.
- Failure criteria.
- Judge rubric or deterministic checks.
- Required evidence fields.
- Thresholds for passing, warning, or failing.
- Notes about what should not be overfit.

## Keep evals connected to failures

Evals can lose value when they drift away from real failures. EDD Platform is intended to connect scenarios, eval contracts, run evidence, and failure packets so that tests remain grounded in observed behavior.
