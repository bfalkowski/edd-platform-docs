This repository exposes the public documentation site under `docs/`. The application source lives in the EDD Platform product repo.

## Documentation preview

From the docs repository root:

```bash
cd docs
npx mint dev
```

The preview usually appears at `http://localhost:3000`.

## Application setup

In the product repo, install web dependencies once:

```bash
cd apps/web
npm install
```

Start local Postgres and the app:

```bash
./scripts/dev.sh
```

The API runs on:

```text
http://127.0.0.1:8001
```

The React console runs on Vite's reported local URL, usually:

```text
http://localhost:5173
```

## Seed the demo projects

After the app is running, seed one or both demo projects from the product repo:

```bash
# Sentiment Observer — canonical walkthrough used by these docs
python scripts/seed_sentiment_observer_demo.py

# Customer Triage — support ticket routing agent
python scripts/seed_customer_triage_demo.py
```

Both scripts create an `AgentDesign`, `AgentVersion`, `Scenario`, `EvalContract`, and supporting artifacts through the API.

## Langfuse (required for Error Analysis)

Langfuse is required for the Error Analysis tab and recommended for the Proof Loop. To run it locally:

```bash
./scripts/dev_langfuse.sh
```

Langfuse runs at `http://localhost:3001`. Local credentials are documented in the product repo README.

Without Langfuse, agent runs in the Proof Loop still execute and produce eval evidence, but no `TRACE_REF` artifacts are created and the Error Analysis tab is unavailable.

## Environment variables

Copy `.env.example` to `.env` and set at minimum:

```bash
ANTHROPIC_API_KEY=sk-ant-...
```

The default model is `claude-haiku-4-5-20251001`. Override with:

```bash
EDD_ANTHROPIC_MODEL=claude-sonnet-4-6
```

- Prefer "intended design" language for planned features.
- Keep terminology aligned with the [Glossary](/reference/glossary).
- Keep screenshots aligned with the Sentiment Observer seeded demo.
