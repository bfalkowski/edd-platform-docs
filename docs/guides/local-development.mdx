This repository exposes the public documentation site under `docs/`. The application source lives in the EDD Platform product repo.

## Documentation preview

From the repository root:

```bash
cd docs
npx mint dev
```

If Mintlify is not installed, `npx` will prompt to download it. The preview usually appears at `http://localhost:3000`.

## Application setup

In the product repo, install web dependencies once:

```bash
cd apps/web
npm install
```

Start local Postgres and the app:

```bash
docker compose up -d postgres
./scripts/dev.sh
```

The API runs on:

```text
http://127.0.0.1:8001
```

The React console runs on Vite's reported local URL, usually:

```text
http://localhost:5173
```

## Seed the canonical demo

After the app is running, seed the Sentiment Observer example from the product repo:

```bash
python scripts/seed_sentiment_observer_demo.py
```

The product repo also includes a customer triage seed script, but Sentiment Observer is the canonical walkthrough used by these docs.

## Optional Langfuse

Local Langfuse is optional. In the product repo:

```bash
./scripts/dev_langfuse.sh
```

Langfuse runs at `http://localhost:3001` with seeded local credentials documented in the product README.

## Contributor checklist

- Keep documentation changes under `docs/` unless the repository adds a broader convention.
- Prefer intended-design language for planned features.
- Keep terminology aligned with the [Glossary](/reference/glossary).
- Keep screenshots aligned with the Sentiment Observer seeded demo.
