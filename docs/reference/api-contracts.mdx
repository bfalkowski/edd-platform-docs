The product repo includes a FastAPI platform API and a generated OpenAPI contract. Public docs stay aligned with the API; the generated contract lives in the product repo.

## Contract principles

The API preserves the product loop and object spine:

```text
Observe -> Analyze -> Measure -> Improve -> Compare -> Gate

AgentDesign -> AgentVersion -> Scenario -> EvalContract -> Run
  -> EvalResult / JudgeOutput -> FailurePacket -> FixProposal
  -> candidate AgentVersion -> Comparison -> GateDecision
```

Error Analysis objects extend the spine:

```text
ReviewCorpus -> ReviewItem -> ReviewAnnotation -> FailureMode
```

## Conventions

- Use stable IDs for all workflow entities.
- Store evidence references separately from summaries.
- Keep external evidence references typed (e.g., `langfuse_trace_id`, `langfuse_score_id`).
- Make comparisons explicit about baseline, candidate, scenarios, and contracts.
- Make gate decisions immutable or auditable after creation.
- Keep the Sentiment Observer seeded demo as ordinary platform records, not hardcoded special cases.
